## Fraxlend Plugin

The Fraxlend plugin (`@iqai/plugin-fraxlend`) provides integration with the Fraxlend protocol, enabling lending and borrowing operations.

### Configuration

```typescript
import { createFraxlendPlugin } from "@iqai/plugin-fraxlend";
import { fraxtal } from "viem/chains";

const fraxlendPlugin = await createFraxlendPlugin({
  chain: fraxtal,
  walletPrivateKey: process.env.WALLET_PRIVATE_KEY,
});
```

### Features

- Lending operations
- Borrowing functionality
- Position management

### Usage Example

```typescript
import { Agent } from "@iqai/agent";
import { createFraxlendPlugin } from "@iqai/plugin-fraxlend";

const agent = new Agent({
  // ... other configuration
  plugins: [createFraxlendPlugin({
    chain: fraxtal,
    walletPrivateKey: process.env.WALLET_PRIVATE_KEY,
  })],
});
```

## Using Multiple Plugins

You can use multiple plugins together to combine their functionality:

```typescript
import { Agent } from "@iqai/agent";
import { createFraxlendPlugin } from "@iqai/plugin-fraxlend";
import { createOdosPlugin } from "@iqai/plugin-odos";
import { fraxtal } from "viem/chains";

async function setupAgent() {
  const fraxlendPlugin = await createFraxlendPlugin({
    chain: fraxtal,
    walletPrivateKey: process.env.WALLET_PRIVATE_KEY,
  });

  const odosPlugin = await createOdosPlugin({
    chain: fraxtal,
    walletPrivateKey: process.env.WALLET_PRIVATE_KEY,
  });

  const agent = new Agent({
    modelProvider: ModelProviderName.OPENAI,
    modelKey: process.env.OPENAI_API_KEY,
    plugins: [fraxlendPlugin, odosPlugin],
    // ... other configuration
  });

  await agent.start();
}
```

## Plugin Configuration

All official plugins share some common configuration parameters:

- `chain`: The blockchain network to operate on
- `walletPrivateKey`: Private key for transaction signing

Additional configuration options may be available depending on the specific plugin.

## Best Practices

1. **Environment Variables**
   - Keep sensitive data like private keys in environment variables
   - Use a `.env` file for local development

2. **Chain Selection**
   - Ensure the selected chain is supported by the plugin
   - Use the correct chain configuration from `viem/chains`
